## Вступ
У даній курсовій роботі розроблено програмну систему для автоматизації процесів обліку товарів у фургоні з продажу кави. Актуальність роботи полягає у необхідності створення ефективного та зручного інструменту для керування асортиментом, відстеження параметрів (ціна, вага, об'єм) та дотримання обмежень місткості та бюджету. Сучасні програмні засоби на базі Java дозволяють створити кросплатформний додаток із зручним графічним інтерфейсом, що робить його придатним для використання на практиці.

---

## 1. Постановка задачі
Функціональні можливості програмної розробки:
- **Облік асортименту:** додавання, редагування, видалення та копіювання різних видів кави (зернова, мелена, розчинна).
- **Обмеження:** контроль завантаження фургону за об'ємом (літри) та бюджетом (грн).
- **Пошук та сортування:** сортування товарів за співвідношенням ціни та ваги, пошук товарів у заданому діапазоні якості.
- **Зберігання даних:** збереження інформації про товари та налаштування фургону у локальній базі даних.
- **Аналітика:** перегляд зведеної статистики та експорт/імпорт даних у форматі CSV.
- **Тип програмного рішення:** настільний застосунок (desktop application).

## 2. Огляд літератури
У процесі розробки проекту було проаналізовано принципи об'єктно-орієнтованого програмування, шаблони проектування (DAO, Singleton) та сучасні підходи до розробки графічних інтерфейсів. Як близькі прототипи розглядалися існуючі системи управління запасами (Inventory Management Systems), однак дане рішення спеціалізоване під конкретну предметну область (продаж кави), що дозволяє врахувати фізичний стан товару (зернова, мелена тощо) та тип пакування. Для реалізації було обрано мову Java завдяки її потужним бібліотекам та платформонезалежності.

- `van_settings`: зберігає поточні обмеження фургону.

## 4. Програмне рішення
Програма написана мовою Java (версія 21).
- **Графічний інтерфейс:** побудований з використанням бібліотеки JavaFX. Головне вікно (`MainWindow`) містить таблицю для відображення товарів, панель фільтрів та статистики.
- **Взаємодія з БД:** реалізована через JDBC. Клас `DatabaseManager` (Singleton) відповідає за встановлення з'єднання та ініціалізацію схеми.
- **Логування та сповіщення:** використовується Log4j2 для запису дій у файл. У разі критичних помилок (наприклад, неможливість підключитися до БД) відправляється email-сповіщення через `SmtpEmailNotifier`.
- **Забезпечення якості:** код покритий unit-тестами з використанням JUnit 5 та Mockito. Досягнуто високий рівень покриття (>90%) бізнес-логіки. Для тестування UI використано TestFX.

## Висновки
В результаті виконання курсової роботи розроблено повнофункціональний програмний застосунок для управління фургоном кави. 
Програма відповідає всім поставленим вимогам:
- Застосовано принципи ООП (спадкування, поліморфізм).
- Реалізовано графічний інтерфейс (JavaFX) та збереження даних (SQLite).
- Забезпечено надійність системи через unit-тестування (JUnit 5, TestFX), логування (Log4j2) та механізм email-сповіщень.
Набуто практичних навичок проектування архітектури, розробки UML-діаграм та використання сучасних технологій Java-розробки.
